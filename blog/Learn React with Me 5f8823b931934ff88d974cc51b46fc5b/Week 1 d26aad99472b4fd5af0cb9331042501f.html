React

React is a JS library for user interfaces. It uses components to build complex UIs

// React component class with JSX
class ShoppingList extends React.Component { // A component takes in parameters, called props
  render() { // return a react element
    return (
      <div className="shopping-list">
        <h1>Shopping List for {this.props.name}</h1>
      </div>
    );
  }
}

React.createElement('div') // creates a div like above instead of jsx

  • React components can have state by setting this.state in their constructors.(it's to remember things)
    • The state is the set of data that is managed by the component.
    • Calling useState(), you will get back a new state variable, and a function that we can call to alter its value.(You can't alter the value of a state variable directly you must use a modifier function)
    • useState() accepts the initial value of the state item and returns an array containing the state variable, and the function you call to alter the state.
    const [count, setCount] = useState(0)
  • If there is a constructor present in the subclass, it needs to first call super() before using "this".
  • In React, it’s conventional to use on[Event] names for props which represent events and handle[Event] for the methods which handle the events.
  • Mutable vs immutable
    • Immutability makes complex features much easier to implement. Avoiding direct data mutation lets us keep previous versions of the game’s history intact, and reuse them later.
    • Detecting changes in immutable objects is considerably easier. If the immutable object that is being referenced is different than the previous one, then the object has changed.
    • The main benefit of immutability is that it helps you build pure components in React. Immutable data can easily determine if changes have been made, which helps to determine when a component requires re-rendering.
