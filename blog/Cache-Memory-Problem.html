Today, I was thinking of going over a cache memory problem. I remember when I was in Computer System how I had to go over that chapter a lot. It wasn’t all on one page but it was chopped up throughout the chapter. And unfortunately, in class, it was even more confusing. Here is a problem and what I was given with it.

Assume the following(given):

  • The memory is byte-addressable.
  • Memory accesses are 1-byte words (not to 4-byte words).
  • Addresses are 10 bits wide.
  • The cache is direct-mapped cache (E = 1), with an 8-byte block size (B=8) and 8 sets (S=8).
    • Direct-mapped means that each memory block is mapped to exactly one block in the cache
  • The cache contents are as shown below
  • Memory Address is 0x4

I have highlighted what’s important for this problem

Problem: Find Cache block Offset, Cache set index, Cache tag, if it’s a Cache hit, and the Cache byte returned in hex.

I will now walk through this problem step by step.

  1. This problem is asking you to find the Cache block Offset, the Cache set index, and the Cache tag. I would suggest you draw a row which represents your 10 bits wide address. The order from left to right is: Cache Tag->Cache set Index->Cache block Offset
  1. You now have to find the size of Cache Tag(CT), Cache set Index(CI), and Cache block Offset(CO). We are going to find CO first. We were given an 8-byte block size. CO = 2b2^b2b﻿ = B = 2b2^b2b﻿ = 8 CO will be b so if we do the math b = 3 CO = 3 Then we want to find CI. We were given 8 sets. CI = 2s2^s2s﻿ = S = 2s2^s2s﻿ = 8 CI will be s, so if we do the math s = 3 CI = 3 Then finally, we want to find CT. CT = m - (b+s) m here is the length of our address which is 10 CT = 10 - (3+3) = 4 To summarize, the size of CO is 3, CI is 3 and CT is 4.
  1. We now want to fill our table
  1. Now we want to fill the table with the address we were given. We want to translate 0x4 into binary. 0x4 = 0100. We will put our binary from right to left.

    We then want to fill the rest with zeros:

  1. The problem asked for Cache block Offset, Cache set index, Cache tag. So now you want to translate the binary back to hex.
  1. Now we want to find if we have a cache hit and what it returns.We need to look at the table given. We need to use CI and CT. CI is 0x0 and CT is 0x0(based from the table we filled). CI will tell us which set, so we are looking at set 0. Then we look at the Tag in that set which is 0x0. The tag and our CT are the same so we have a match. So yes we have a cache hit.
  1. Now we want to see what it returns. We need to use our CO which is 0x4(Decimal =4). So in the same set as step 6 (set 0), we want to look at our Data. We are going to count till we get to the 4th number(starting at 0x4d which is at the zeroth position). The number at the 4th position is 0x40 which is the Cache byte returned.
  1. Summary CO = 0x4 CI = 0x0 CT = 0x0 Cache hit = yes Cache byte returned = 0x40
